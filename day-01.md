
# CSE English Vocabulary - Day 01

| Word | Meaning |
|--------|---------|
| Algorithm | Problem-solving steps |
| Function | Reusable block of code |
| Variable | Named storage location |
| Parameter | Function input definition |
| Argument | Actual value passed to function |
| Return | Send result back |
| Compile | Convert source code to executable |
| Execute | Run a program |
| Debug | Find and fix errors |
| Syntax | Rules of a programming language |
| Logic | Reasoning behind a solution |
| Data | Information |
| Structure | Organization or arrangement |
| Array | Collection of similar data |
| Index | Position in an array |
| Loop | Repetition construct |
| Condition | Decision-making rule |
| Iterate | Repeat through items |
| Declare | Create a variable |
| Initialize | Assign initial value |
| Instance | Specific object created from class |
| Object | Entity containing data and methods |
| Class | Blueprint for objects |
| Method | Function inside a class |
| Attribute | Property of an object |
| Inherit | Acquire features from parent class |
| Encapsulation | Hide internal details |
| Abstraction | Show essential features only |
| Interface | Contract for interaction |
| Framework | Development structure |
| Library | Collection of reusable code |
| Dependency | Required external component |
| Repository | Project storage location |
| Commit | Save changes in Git |
| Branch | Separate line of development |
| Merge | Combine branches |
| Clone | Copy a repository |
| Deploy | Publish application |
| Server | Service provider |
| Client | Service consumer |
| Request | Ask for data |
| Response | Return data |
| Database | Data storage system |
| Query | Request information from database |
| Authentication | Verify identity |
| Authorization | Verify permissions |
| Secure | Protected from threats |
| Optimize | Improve performance |
| Scalable | Able to grow efficiently |
| Maintainable | Easy to update and manage |
